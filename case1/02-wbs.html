WBS (Work Breakdown Structure)

착수 → 분석 → 설계 → 개발 → 테스트 → 이행의 6단계 일정 계획을 주차 단위로 구조화하여, RFP 기반 프로젝트의 일정/역할/선행관계를 명확히 관리하기 위한 기준안이다.

1. WBS 개요

WBS는 프로젝트 범위를 실행 가능한 작업 단위로 분해하고, 일정·담당·선행관계를 연결하는 관리 도구다. RFP 기반 사업에서는 제안 단계의 공수 추정과 실제 수행 단계의 관리 기준이 연결되어야 하므로, 상위 단계부터 세부 작업까지 일관된 체계를 유지하는 것이 중요하다.

아래 예시는 16주 프로젝트를 기준으로 작성되었으며, 규모가 큰 사업은 각 Phase 내부 작업을 더 세분화하고 마일스톤 승인 절차를 추가한다.

2. Phase Overview

Phase 단계명 설명
1.0착수/관리프로젝트 통제 체계와 일정, 품질, 리스크 관리 기준을 확립한다.
2.0분석요구사항, 데이터, 화면, 인터페이스를 분석해 설계 입력값을 정리한다.
3.0설계시스템 구조, DB, API, UI의 상세 설계를 확정한다.
4.0개발DB, BE, FE, 인프라를 구현하고 통합 가능한 상태로 만든다.
5.0테스트단위/통합/사용자 시나리오 검증과 결함 조치를 수행한다.
6.0이행/안정화운영 전환, 사용자 교육, 초기 장애 대응과 안정화를 수행한다.

4. 일정 운영 포인트

  • 프로젝트 관리(1.0)는 전 기간에 걸쳐 지속되며, 상세 작업은 각 Phase와 병행된다.
  • 분석과 설계는 완전히 분리하기보다 일부 중첩 운영이 가능하나, 승인 기준은 분명히 해야 한다.
  • 개발 단계는 DB/BE/FE/Infra를 병렬 추진하되, 통합 환경 구성 시점을 명확히 두어야 테스트 지연을 줄일 수 있다.
Example

공공사업에서 발주처 중간보고가 8주차에 예정되어 있다면, Phase 3 설계 산출물과 일부 시제품 화면을 해당 주차에 맞춰 재배치하는 식으로 WBS를 조정할 수 있다.

위 정보를 기준으로 착수 → 분석 → 설계 → 개발 → 테스트 → 이행의 6단계 WBS를 작성해줘.
조건:
1) 표 컬럼은 WBS Code, 작업명, 기간(주), 시작주, 종료주, 담당, 선행작업
2) 프로젝트 관리 Phase는 별도로 두고 PM-001 ~ PM-005 수준의 관리 작업 포함
3) 분석/설계/개발/테스트/이행 단계별 세부 작업을 작성
4) 마일스톤은 ◆ 표시를 포함
5) HTML 테이블에 바로 넣을 수 있도록 간결한 문장으로 정리

기간과 인력 규모를 함께 입력하면 AI가 병렬 개발 가능 여부와 현실적인 주차 배분을 더 잘 판단한다.
