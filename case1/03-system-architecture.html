시스템 구성안

전체 시스템 아키텍처 구성도와 인프라 설계를 정리하여, 기능 요구사항과 비기능 요구사항을 동시에 만족하는 구조를 설명하기 위한 기준 문서다.

1. 아키텍처 개요

시스템 구성안은 서비스 흐름과 운영 구조를 동시에 설명해야 한다. 특히 RFP 기반 사업에서는 성능, 보안, 가용성, 확장성, 운영 효율을 설계 원칙으로 명확히 제시해야 심사와 수행 모두에 유리하다.

설계 원칙
  • 계층 분리: Presentation, API, Application, Data, Infrastructure 역할을 명확히 분리
  • 고가용성: 단일 장애 지점 최소화와 이중화 구성 확보
  • 보안 우선: 외부 공개 구간과 내부 업무 구간 분리, 최소 권한 원칙 적용
  • 운영 자동화: 배포, 모니터링, 장애 대응 체계의 자동화 강화

계층 구성요소 역할 검토 대상 기술

Application업무 서비스, 배치, 연계 모듈비즈니스 로직 처리, 외부 시스템 연계, 비동기 처리Spring Boot, NestJS, FastAPI, Kafka
DataRDBMS, Cache, Object Storage업무 데이터 저장, 조회 성능 향상, 파일 저장PostgreSQL, MySQL, Redis, S3
InfrastructureLB, VM/Container, Network, CI/CD실행 환경 제공, 배포 자동화, 네트워크 통제Kubernetes, ECS, Jenkins, GitHub Actions
MonitoringAPM, Metrics, Log, Alert시스템 상태 관측, 장애 탐지, 운영 분석Datadog, Prometheus, Grafana, ELK

3. 시스템 구성도

Mermaid Flowchart
flowchart TD
    U[사용자] --> CDN[CDN / 정적 리소스]
    U --> WAF[WAF]
    WAF --> LB[Load Balancer]
    LB --> WEB[Presentation Server]
    WEB --> GW[API Gateway]
    GW --> APP1[Application Server A]
    GW --> APP2[Application Server B]
    APP1 --> CACHE[Redis Cache]
    APP2 --> CACHE
    APP1 --> DB[(PostgreSQL Cluster)]
    APP2 --> DB
    APP1 --> EXT[외부 연계 시스템]
    APP2 --> EXT
    APP1 --> LOG[Central Logging]
    APP2 --> LOG
    LOG --> MON[Monitoring / Alerting]

위 구성은 외부 사용자의 요청을 WAF와 LB를 통해 분산 처리하고, Application 계층의 수평 확장과 Cache 활용으로 응답시간을 안정적으로 유지하는 구조를 보여준다.

4. 네트워크/인프라 구성

  • DMZ 분리: 외부 노출 구간은 WAF, LB, Reverse Proxy 등 최소 구성만 위치시키고, 업무 서버와 DB는 내부망에 배치한다.
  • 로드밸런싱: 무중단 운영을 위해 L7 로드밸런싱과 Health Check 기반 자동 제외 정책을 적용한다.
  • 서버 구성: Web/App 분리 또는 Container 기반 Microservice 구성을 선택하되, 운영 복잡도를 함께 검토한다.
  • 스토리지: 정적 파일과 백업 파일은 Object Storage, 트랜잭션 데이터는 고가용성 RDBMS에 저장한다.
  • DR 전략: 정기 백업, 이중화, RPO/RTO 기준 정의, 재해복구 모의훈련 계획을 포함한다.
DR은 단순 백업이 아니라 “복구 시간과 데이터 손실 허용 범위”를 수치로 정의해야 의미가 있다. RFP에 24x7 운영이 있으면 DR 수준도 함께 상향 검토해야 한다.

6. 보안 아키텍처

보안은 별도 부가 기능이 아니라 전체 아키텍처의 기본 전제다. 인증·인가, 네트워크 접근제어, 데이터 암호화, 로그 감사, 비밀정보 관리, 취약점 대응이 함께 설계되어야 한다.

  • 인증/인가: SSO, MFA, Role 기반 권한 정책
  • 네트워크 보안: Security Group, ACL, VPN, Bastion Host
  • 데이터 보안: 개인정보/민감정보 암호화, Key 관리 분리
  • 감사 추적: 관리자 행위 로그, API 접근 로그, 변경 이력 보관
  • 운영 보안: 시크릿 매니저, 이미지 취약점 스캔, 배포 승인 정책
