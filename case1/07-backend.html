Back-End 설계

BE 아키텍처, API 명세, 인증/인가 체계, 비즈니스 로직 설계를 정리하여 안정적인 서버 구조와 연계 개발 기준을 정의하는 문서입니다.

1. BE 아키텍처 개요

본 프로젝트의 Back-End는 Layered Architecture를 기본 원칙으로 하며, Controller → Service → Repository 계층으로 책임을 분리합니다. Controller는 요청/응답과 입력 검증을 담당하고, Service는 비즈니스 규칙과 트랜잭션을 제어하며, Repository는 데이터 영속성 처리를 담당합니다.

이 구조는 기능 확장 시 계층별 영향 범위를 최소화하고, 테스트와 유지보수성을 높이는 데 유리합니다. 공통 인증, 예외처리, 로깅, API 문서화는 횡단 관심사로 별도 공통 모듈화합니다.

5. 인증/인가 흐름

인증은 JWT 기반으로 처리합니다. 사용자가 로그인하면 서버는 Access Token과 Refresh Token을 발급하고, 이후 요청은 Authorization 헤더의 Bearer Token을 통해 인증됩니다. Access Token 만료 시 Refresh Token을 통해 재발급 절차를 수행합니다.

인가 체계는 RBAC를 적용합니다. 예를 들어 Admin은 사용자 관리, 공지사항 관리, 권한 부여 기능에 접근 가능하며, 일반 User는 게시판 조회/등록, 마이페이지 기능에 접근 가능합니다. 각 API 엔드포인트는 역할별 접근 제어 정책을 명시적으로 선언합니다.

인증 실패(401)와 권한 부족(403)은 반드시 구분하여 응답해야 하며, 감사 로그에는 사용자 ID, 권한, 요청 자원 정보를 함께 남깁니다.
