산출물별 AI 프롬프트 예시

주요 분석/설계 산출물을 빠르게 초안 작성할 수 있도록 8개 핵심 문서별 AI 코드어시스턴트 프롬프트 템플릿을 정리한 문서입니다.

1. 요구사항 정의서 프롬프트

요구사항 정의서
다음 RFP와 프로젝트 개요를 바탕으로 요구사항 정의서를 작성해줘.

[입력]
- 프로젝트 목적
- 사용자 유형
- 핵심 기능
- 비기능 요구사항
- 제약사항

[출력 조건]
- 기능 요구사항 / 비기능 요구사항을 구분할 것
- 요구사항 ID를 부여할 것
- 우선순위, 수용 기준, 관련 부서를 포함할 것
- 표 형식과 설명 문단을 함께 작성할 것

실무 문서 수준의 한국어로 정리해줘.
RFP 본문과 고객 인터뷰 내용을 함께 넣으면 누락 없는 요구사항 초안을 만드는 데 유리합니다.

2. 유스케이스 명세서 프롬프트

유스케이스 명세서
다음 시스템 기능 목록을 바탕으로 유스케이스 명세서를 작성해줘.

[입력]
- 액터 목록
- 주요 기능 목록
- 업무 시나리오

[출력 조건]
- 유스케이스 ID, 명칭, 액터, 사전조건, 기본 흐름, 예외 흐름, 사후조건을 포함할 것
- 관리자/사용자 역할 차이를 반영할 것
- 주요 화면 또는 API 연계를 함께 언급할 것

표와 단계별 서술을 포함한 한국어 문서로 작성해줘.
복잡한 업무 프로세스는 예외 흐름과 승인/반려 조건을 함께 넣어야 결과 품질이 높아집니다.

3. 화면 정의서 프롬프트

화면 정의서
다음 요구사항과 사용자 시나리오를 기반으로 화면 정의서를 작성해줘.

[입력]
- 화면 목록
- 사용자 역할
- 주요 입력/출력 데이터
- 화면 간 이동 흐름

[출력 조건]
- 화면 ID, 화면명, 경로, 권한, 주요 기능을 표로 정리할 것
- 입력 항목, 검증 규칙, 버튼 동작을 포함할 것
- 목록/상세/등록/수정 화면의 관계를 설명할 것

실무 설계서 형식으로 정리해줘.
와이어프레임 초안이 있으면 입력 항목과 버튼 동작 정의가 훨씬 구체적입니다.

4. 데이터베이스 설계서 프롬프트

데이터베이스 설계서
다음 업무 요구사항을 기반으로 데이터베이스 설계서를 작성해줘.

[입력]
- 주요 엔터티
- 데이터 항목 정의
- 관계 정보
- 보존 정책 및 개인정보 항목

[출력 조건]
- 개념/논리/물리 데이터 모델 관점으로 설명할 것
- 테이블명, 컬럼명, 타입, PK/FK, NULL 여부를 표로 정리할 것
- 인덱스, 정합성, 이력 관리 정책을 포함할 것

한국어 실무 문서 형식으로 작성해줘.
개인정보와 감사 로그 요구사항을 함께 주면 물리 모델 설계에 필요한 컬럼을 더 잘 반영할 수 있습니다.

5. API 명세서 프롬프트

API 명세서
다음 기능 요구사항을 기반으로 RESTful API 명세서를 작성해줘.

[입력]
- 리소스 목록
- 사용자 역할 및 권한
- CRUD 범위
- 인증 방식

[출력 조건]
- 메서드, URI, 설명, 요청 파라미터, 요청 바디, 응답 예시, 에러 코드를 포함할 것
- OpenAPI 스타일로 정리할 것
- 인증/인가, 페이징, 검색 조건을 반영할 것

실제 개발자 전달용 문서 형식으로 작성해줘.
가능하면 리소스별 샘플 JSON을 함께 입력해 응답 구조 일관성을 높이는 것이 좋습니다.

6. 시스템 아키텍처 설계서 프롬프트

시스템 아키텍처 설계서
다음 프로젝트 정보를 기반으로 시스템 아키텍처 설계서를 작성해줘.

[입력]
- 서비스 구성 요소
- 사용자 규모 및 트래픽 가정
- 인프라 환경
- 외부 연계 시스템
- 보안/가용성 요구사항

[출력 조건]
- 전체 시스템 구성도 설명
- Web/App/DB/연계/보안/운영 구조를 계층별로 정리
- 이중화, 백업, 모니터링, 배포 구조를 포함
- 기술 선택 이유와 장단점을 함께 기술

한국어 설계서 형식으로 작성해줘.
운영 환경이 온프레미스인지 클라우드인지 명확히 넣어야 현실적인 아키텍처 초안을 얻을 수 있습니다.

7. 테스트 계획서 프롬프트

테스트 계획서
다음 프로젝트 일정과 기능 범위를 기준으로 테스트 계획서를 작성해줘.

[입력]
- 개발 범위
- 릴리즈 일정
- 테스트 환경
- 품질 목표

[출력 조건]
- 단위/통합/시스템/UAT 테스트 범위를 구분할 것
- 테스트 조직, 역할, 일정, 진입/종료 기준을 포함할 것
- 결함 관리 프로세스와 보고 체계를 설명할 것

실무 PM/QA 문서 수준의 한국어로 정리해줘.
배포 일정과 주요 마일스톤을 함께 입력하면 테스트 일정안이 실제 프로젝트 일정과 더 잘 맞습니다.

8. 보안 설계서 프롬프트

보안 설계서
다음 시스템 정보를 기반으로 보안 설계서를 작성해줘.

[입력]
- 인증 방식
- 권한 체계
- 개인정보 처리 항목
- 외부 연계 구간
- 감사 및 로그 요구사항

[출력 조건]
- 인증, 인가, 암호화, 접근통제, 로그/감사, 취약점 대응 항목을 포함할 것
- 구간별 보안 통제 방안을 표로 정리할 것
- 운영 보안과 개발 보안을 구분할 것
- 관련 법규 및 가이드라인 고려사항을 반영할 것

실무 심사 대응 가능한 한국어 문서 형식으로 작성해줘.
개인정보, 접속기록, 관리자 행위 로그 요구사항을 상세히 줄수록 심사 대응형 문서 초안에 유리합니다.
