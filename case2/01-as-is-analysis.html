현행(AS-IS) 시스템 분석

기존 시스템 구조, 소스코드, DB, 인프라를 전수 조사하여 전환 범위와 난이도의 근거를 만드는 분석 문서입니다.

1. 현행 분석 개요

현행 분석은 단순 인벤토리 수집이 아니라 전환 영향도, 재사용 가능성, 제거 대상, 리스크를 함께 식별하는 과정이다. 분석 기준은 문서가 아니라 실제 운영 자산을 우선하며, 공통 아키텍처 문서, FE 화면, BE 서비스, DB 데이터, Infra 배포 환경을 같은 기준으로 정리해야 한다.

핵심 원칙은 다음과 같다.

  • 현행 문서보다 실행 중인 시스템과 실제 소스 저장소를 우선 확인한다.
  • 기능 목록과 데이터 흐름을 연결하여 전환 누락을 방지한다.
  • 운영 중 장애 이력, 성능 병목, 수작업 절차까지 함께 수집한다.
  • 재개발, 재사용, 래핑, 폐기 대상을 초기에 구분한다.

4. DB 분석 항목

DB 분석은 단순 ERD 확인을 넘어 전환 가능성과 데이터 품질을 동시에 검토해야 한다.

스키마 구조

핵심 엔터티, PK/FK 관계, 정규화 수준, 파티셔닝, 인덱스 전략을 확인한다.

데이터 품질

중복 데이터, NULL 정책 불일치, 코드값 관리 방식, 삭제 데이터 보존 규칙을 점검한다.

배치 및 인터페이스

야간 배치, ETL, 외부 시스템 적재/수신 테이블, 대량 처리 구간을 식별한다.

DBMS 종속성

프로시저, 함수, 트리거, 힌트, 벤더 전용 SQL 문법 등 이식성에 영향을 주는 요소를 분류한다.

5. 현행 분석 체크리스트

  • 운영 중인 실제 소스 저장소와 배포본 버전 차이를 확인했는가?
  • 전 메뉴/화면과 배치/관리 기능까지 목록화했는가?
  • DB 오브젝트와 데이터 규모, 증가율을 수치로 정리했는가?
  • 외부 연동 실패 시 재처리 방식까지 파악했는가?
  • 상용 솔루션 및 오픈소스 라이선스 제약을 확인했는가?
  • 성능 병목과 장애 다발 구간을 운영 이력 기반으로 추출했는가?
  • 전환 제외 대상과 폐기 후보를 구분했는가?
