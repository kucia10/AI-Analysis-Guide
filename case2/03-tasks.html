개발 파트별 Task 정의 (컨버팅)

컨버팅 프로젝트에 특화된 PM, FE, BE, DB, Infra 파트별 세부 작업과 산출물을 정의합니다.

1. PM Task

Task ID 작업명 상세 내용 산출물
PM-01PM 프로젝트 착수이해관계자 정의, 목표/범위 공유, 의사결정 체계 구성착수보고서, RACI
PM-02PM 현행분석 계획분석 대상, 일정, 인터뷰 계획, 자료 요청 목록 수립현행분석 계획서
PM-03PM 전환 범위/전략 수립전환 유형 선정, 범위/제외 범위, 단계별 접근 전략 정리전환 전략서
PM-04PM 리스크 관리핵심 리스크 식별, 대응/완화 계획, 주간 점검 운영리스크 레지스터
PM-05PM 전환 교육 계획운영자/사용자 대상 교육 범위, 일정, 자료 계획 수립교육 계획서
PM-06PM 형상/이행 관리브랜치/배포 기준, 변경통제, 컷오버 승인 절차 관리형상관리 계획, 이행계획서

2. FE Task

Task ID 작업명 상세 내용 산출물
FE-01FE 현행 화면 분석메뉴, 화면 흐름, 공통 컴포넌트, 사용자 액션 패턴 분석화면 목록, 화면 분석서
FE-02FE UI/UX 개선 도출현행 불편 요소와 접근성/반응형 개선 항목 도출UI/UX 개선안
FE-03FE 신규 FE 아키텍처 설계라우팅, 상태관리, 공통 모듈, API 연동 구조 설계FE 아키텍처 설계서
FE-04FE 디자인 시스템 구축토큰, 공통 컴포넌트, 화면 규칙, 품질 기준 정의디자인 시스템 가이드
FE-05FE 화면 전환 개발우선순위 화면부터 단계적으로 신규 구조로 개발 및 연결전환 화면, 컴포넌트 목록
FE-06FE FE 검증기능 동등성, 브라우저 호환성, 접근성, UI 회귀 검증 수행FE 테스트 결과서

3. BE Task

Task ID 작업명 상세 내용 산출물
BE-01BE 현행 비즈니스 로직 분석서비스, 배치, 프로시저에 분산된 업무 규칙과 예외 흐름 정리로직 분석서
BE-02BE API 매핑 분석현행 인터페이스와 신규 API 구조의 매핑 관계 정의API 매핑 매트릭스
BE-03BE 신규 BE 아키텍처 설계계층 구조, 모듈 경계, 트랜잭션, 예외 처리 정책 설계BE 아키텍처 설계서
BE-04BE 비즈니스 로직 전환핵심 도메인 로직을 신규 구조로 이식하고 테스트 코드 작성전환 서비스 모듈
BE-05BE 인터페이스 전환외부 연동, 배치, 메시지, 파일 연계를 신규 방식에 맞게 재구성인터페이스 전환서
BE-06BE API 역호환성 검증기존 클라이언트와 연동 시스템 영향도를 검증하고 보완호환성 검증 결과서

4. DB Task

Task ID 작업명 상세 내용 산출물
DB-01DB 현행 DB 스키마 분석테이블, 인덱스, 제약조건, 프로시저, 배치 테이블 구조 분석DB 분석서
DB-02DB DBMS 호환성 분석벤더 종속 문법, 타입 매핑, 함수/프로시저 이식성 검토호환성 분석서
DB-03DB 신규 스키마 설계정규화, 성능, 감사, 이력, 코드 체계를 반영한 신규 스키마 설계TO-BE ERD
DB-04DB 데이터 마이그레이션 설계이관 범위, 절차, 검증 기준, 컷오버 방식 정의마이그레이션 설계서
DB-05DB 마이그레이션 스크립트 개발DDL/ETL/정합성 체크 SQL과 반복 실행 스크립트 작성이관 스크립트
DB-06DB 데이터 정합성 검증건수, 합계, 키 무결성, 표본 비교 기준으로 검증 수행정합성 검증 결과서

5. Infra Task

Task ID 작업명 상세 내용 산출물
IF-01Infra 현행 인프라 분석서버, 네트워크, 보안장비, 배포 구조, 운영 절차 현황 파악인프라 현황서
IF-02Infra 신규 인프라 설계가용성, 확장성, 보안, 운영 표준을 반영한 TO-BE 설계인프라 구성안
IF-03Infra CI/CD 파이프라인 구축빌드, 테스트, 배포, 승인, 롤백 절차 자동화파이프라인 구성서
IF-04Infra 이중 환경 운영현행/신규 병행 운영 환경을 구성하고 전환 리허설 수행병행 운영 계획서
IF-05Infra 컷오버 계획전환 당일 절차, 체크포인트, 롤백 조건, 승인 체계를 정리컷오버 계획서
