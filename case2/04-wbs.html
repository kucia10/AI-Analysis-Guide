WBS (컨버팅 프로젝트)

현행분석부터 전환 설계, 개발, 테스트, 컷오버 및 안정화까지 이어지는 컨버팅 프로젝트의 표준 일정 구조입니다.

1. 컨버팅 WBS 개요

컨버팅 프로젝트는 새로 만드는 프로젝트와 달리 현행 분석 정확도와 전환 리허설의 비중이 높다. 따라서 일정은 분석과 설계에 충분한 시간을 확보하고, 테스트 단계에 데이터 정합성과 기능 동등성 검증을 포함하도록 구성한다.

2. Phase Overview

  • Phase 1: 프로젝트 관리
  • Phase 2: 현행 분석
  • Phase 3: 전환 설계
  • Phase 4: 전환 개발
  • Phase 5: 전환 테스트
  • Phase 6: 컷오버/안정화

4. 마일스톤

표의 ◆ 표시는 주요 마일스톤이다. 권장 마일스톤은 현행 분석 완료, 전환 설계 승인, 전환 개발 완료, 통합 테스트 승인, 컷오버 승인, 안정화 종료이며, 각 시점마다 Go/No-Go 의사결정을 수행한다.
