DB 마이그레이션 설계

DBMS 전환, 스키마/데이터 변환, 정합성 검증 계획을 통합적으로 정리하는 전환 설계 문서입니다.

1. DB 마이그레이션 개요

DB 마이그레이션은 컨버팅 프로젝트에서 가장 리스크가 큰 영역입니다. 데이터 손실, 타입 불일치, 배치 중단, 프로시저 오동작은 서비스 장애로 직결되므로 사전 분석, 반복 리허설, 정합성 검증 기준을 반드시 선행해야 합니다.

DB 전환은 단순한 데이터 복사가 아니라 스키마 구조, SQL 호환성, 성능 튜닝 포인트, 배치 및 인터페이스 영향도까지 함께 다루는 작업입니다. 특히 운영 데이터가 누적된 레거시 환경일수록 예외 데이터와 히스토리 데이터의 처리 기준을 먼저 확정해야 합니다.

2. 마이그레이션 프로세스

단계 주요 활동 핵심 산출물
단계 1. 현행 분석 DB 카탈로그 수집, 테이블/뷰/인덱스/시퀀스/프로시저 등 객체 전수 조사 DB 객체 목록, 용량/증감 추이, 의존성 맵
단계 2. 호환성 분석 데이터타입 매핑, SQL 문법 차이, 함수/패키지 호환성 검토 호환성 이슈 리스트, 변환 규칙서
단계 3. 스키마 변환 DDL 변환, PK/FK/제약조건 이관, 인덱스 재설계 TO-BE DDL, 인덱스 설계서
단계 4. 데이터 변환 ETL 스크립트 작성, 코드값 표준화, 데이터 클렌징 수행 적재 스크립트, 클렌징 규칙, 이행 배치 절차
단계 5. 프로시저 전환 PL/SQL → PL/pgSQL 변환 또는 애플리케이션 로직으로 이관 프로시저 전환 목록, 이관 기준서
단계 6. 마이그레이션 실행 전체 이관, 병렬 적재, 인덱스 생성, 통계정보 갱신 실행 로그, 이행 결과 리포트
단계 7. 검증 건수/합계/참조무결성/업무 시나리오 기반 정합성 확인 정합성 검증 결과서, 보완 이슈 리스트

4. 데이터 정합성 검증 체크리스트

  • 테이블별 총 건수 비교 및 증감 사유 확인
  • 금액/수량 주요 컬럼의 합계 비교
  • 필수값(Null 불가 컬럼) 누락 여부 점검
  • 유니크 키/업무 키 중복 여부 확인
  • 참조무결성(FK 기준 부모/자식 데이터) 점검
  • 표본 샘플링을 통한 원천/대상 데이터 값 비교
  • 주요 업무 시나리오 기준 비즈니스 규칙 검증

5. 검증 SQL 예시

Example
-- 1) 건수 비교
SELECT 'CUSTOMER' AS table_name, COUNT(*) AS cnt FROM customer;

-- 2) 금액 합계 비교
SELECT SUM(order_amount) AS total_amount FROM orders;

-- 3) 필수값 누락 점검
SELECT COUNT(*) AS null_cnt
FROM orders
WHERE customer_id IS NULL;

-- 4) 참조무결성 점검
SELECT COUNT(*) AS orphan_cnt
FROM order_item oi
LEFT JOIN orders o ON oi.order_id = o.order_id
WHERE o.order_id IS NULL;
